# Testing Strategy

Universal verification principles. Project-specific test tooling — runner, framework, coverage setup, CI integration — is documented in `docs/ai/testing.md`, generated by `/adopt-repo-docs`.

## Preferred layers

- **Unit tests** — domain rules, utilities, pure functions, data transformations.
- **Integration tests** — routes, services, repositories, jobs, and infrastructure-touching code.
- **End-to-end tests** — critical user paths only; keep the suite small and stable.

## Minimum expectation for meaningful feature work

- Adjust existing tests when behavior changes.
- Add tests for newly introduced behavior.
- Document deliberately untested areas when they remain — do not silently leave gaps.

## Verification proportionality

Match verification scope to risk tier:
- **L1**: targeted static checks only.
- **L2**: relevant unit and integration tests.
- **L3/L4**: full relevant verification matrix with explicit residual risk statement.
