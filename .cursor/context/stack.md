# Stack Baseline

Universal operational assumptions for all repositories using this kit.
Project-specific runtime, framework, data, and tooling details live in `docs/ai/stack.md` — generated by `/adopt-repo-docs`.

## Universal assumptions

- Verification flow matters more than agent autonomy; prefer explicit quality gates over speed.
- Commit hygiene matters: one logical change per commit, conventional commit messages.
- Local Cursor-only scripts live under `.cursor/local/bin/` and are gitignored.
- Kit-internal documentation lives under `.cursor/docs/`; project knowledge lives in `docs/ai/` and `docs/`.
- Prefer explicit types; avoid dynamic equivalents unless the task explicitly requires them.

## Project stack

See `docs/ai/stack.md` for this repository's specific runtime, framework, data, storage, and testing choices.
If `docs/ai/stack.md` is missing, run `/adopt-repo-docs` to generate it.
